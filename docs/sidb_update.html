Methods

(static) records(storeName, indexName, query, objectValues, errorCallbackopt)

Source:
  • sidb.js, line 1553
Adds the task "update record/s" to the task queue.
Example
var mydb = new sidb();

// An example of object stored in the object store
//
var person = {
    name: 'Peter',
    age: 32,
    salary: 1500
}


//
// Changes the age and salary using an index named "names" with keypath 'name' as query.
//
mydb.update.records(
    'objectStoreName',
    'names',
    'Peter',
    {age: 33, salary: 1650},
    myErrorCallback
);


//
// Increases the salary in 200 to all persons with age > 40 and salary < 1000 using a conditionObject array as query.
// We can send a function to the property salary on selected records.
//
mydb.update.records(
    'objectStoreName',
    null,
    'age > 40 & salary < 1000',
    {salary: function(oldSalary){
        return oldSalary + 200;
        };
    },
    myErrorCallback
);


// Execs all pending tasks.
//
mydb.execTasks();


// Optional callback function to process errors
//
function myErrorCallback(event){
    console.log(event.target.error);
};
Parameters:
Name Type Attributes Description
storeName string Object store name.
indexName string | null Index name. If is null then no index is used (It is usually slower).
query string String that contains a query. Example of valid querys: c > 20 // Simple query c > 10 & name='peter' // Query with 2 conditions (c > 10 && name = 'peter') // Same effect that prev query (&=&& and |=||). (a > 30 & c <= 10) || (b = 100 || d < 50) // 2 conditions blocks 'peter' // Single value always refers to the index keypath.
objectValues object Object with the new values. The values not only can be a single value, it can be a function that receives the old value and returns a new value. (Example: objectValues = {property1:'value1', property4: value4, property6: function(oldValue){return oldValue + 100;}})
errorCallback function <optional>
 Function called on error. Receives event parameter.
